# Dark Filaments — Project Charter

> Idle clicker that disguises a meditation on entropy, loss, and the cost of consolidation. The player thinks they're building a galactic empire. They're causing the heat death of the universe. The single design idea worth protecting through every revision is the **emotional curve**: up → up → up → peak → first thinning → cold → silent.

## Load-bearing rules

These are the current load-bearing constraints — the rules that protect the project's design intent through every revision. The failure modes for this project are subtle; these rules are the guardrails against drift.

**The project is still in early design.** Any of these rules can be changed if the change would improve the overall experience — but rule revisions are deliberate user-facing decisions, never silent agent calls. When a proposal would benefit from revising a rule, route it through `rules-guardian` for a rigorous vetting pass: identify the rule, articulate the proposed change, name the design tradeoff (what's gained, what's at risk), reference the original rationale, surface to the user. The user decides; if accepted, CLAUDE.md is updated to reflect the new state.

Within a single session, treat the rules below as hard constraints. Across sessions and design iterations, treat them as the current best thinking.

- **Real cosmology only.** Every upgrade name, every named structure, every term is real. No invented vocabulary, ever. Poetic license in arrangement, not in invention.
- **First-person plural narrator throughout, until the endgame.** The narrator is "we" until the last connection breaks, then shifts to "I." No exceptions.
- **No exclamation points.** Anywhere. Not in flavor text, not in achievement popups, not in code-facing text destined for the player.
- **No second person.** The narrator never says "you." The player is part of the "we" or not addressed at all.
- **Two-voice UI.** Dark Filaments has two voice registers. **Fading prose** is the narrator ("we" through Act 1 and most of Act 2; "I" in the endgame). **Persistent prose** is clinical — real-cosmology terminology, no first or second person, no editorializing, no agency. The two registers do not cross. Exceptions: the endgame "I" lines and the Inventory's banner-plus-preserved-quotations are the narrator's persistent artifact, post-fade. The *Causal Connections* label is clinical from the moment it appears at the first named-connection break; the narrator's confession lands in the fading pivot line that fires in the same beat, not in the label. The click verb at the tap point (*Pull / Bind / Consolidate / Hold / Reach*) is clinical — physics labels for what the tap does. Existing narrator-voice upgrade flavor text does not get rewritten; it relocates to a one-time ephemeral fade-in on first purchase of each upgrade (per-upgrade per-game; engine dedups against idle interjection scheduling). The persistent upgrade card carries only the clinical one-line description.
- **No metaphors that aren't physics.** The grief is in the description, not in figurative language.
- **No log, no scrollback during gameplay (player-facing only).** Player-facing flavor text fades and is not stored while the universe is live — the central theme working as designed. Known exceptions: the endgame "I" lines (persist forever); the **Inventory artifact** (final-tier reveal, chronological scroll of all lost structures with their break-lines preserved, persistent past the final fade, returnable to indefinitely); and **named one-shot structures appearing in the 3D scene on purchase**. The scene is the universe — it is not a log. Structures disappearing on named-connection breaks are the loss itself, not a record of it. The Inventory is the only post-completion textual record and is canonical. **Development tooling is unambiguously exempt:** dev consoles, instrumentation logs, simulator traces, debug overlays, parameter editors, and any other surface the player never sees in the shipped game are not subject to this rule. Flag any *player-facing in-game* history/log/replay surface before building it.
- **Mass is the only currency the player spends.** Consolidation is a progress meter that gates tier transitions. Not grindable, not spendable beyond gates. *"It is cookies."*
- **Stats carry over between tiers.** Mass is denominated in solar masses (M☉). The mass counter is a real-scale reading from ~1 M☉ at the first tier (Solar System) through ~5×10²² M☉ at the final tier (Causal Horizon). All numerical references in narrator prose use this unit, including "solar masses," "suns," and "stars" as appropriate to the prose. Mass, MPS, MPC, AC/s, and all multipliers from purchased upgrades carry forward when advancing tiers. Older upgrades cannot be repurchased; their effects do not reset. **Once an upgrade is purchased, it produces its contribution for the entire game — flat additive output keeps firing across all subsequent tiers (becoming negligible relative to later-tier scale, but never zero). Global multipliers (`allMps`, `allMpc`, `allAps`) compound across tiers and amplify every contribution currently active: if T1 grants ×2 and T2 grants ×2, every owned stackable's output is multiplied by ×4 from that moment forward. Intra-upgrade exponents (`selfMps` and the like — e.g., Stellar Kinematics' ×1.115 per level) stay local to their own upgrade by construction, but the resulting per-upgrade value is still amplified by every global multiplier owned at any tier.** This mechanically encodes the design's central feedback loop — "the better you play, the faster the universe ends." Early optimization compounds into faster late-tier progression, which compounds into a shorter total universe lifespan.
- **The hidden number is shown from minute one, unlabeled, and does not change during Act 1.** The label *Causal Connections* fades in only at the first named-connection break. The naming IS the reveal. **The number itself sits static throughout Act 1** — present, unexplained, accumulating perceptual weight without ever moving. Ticking begins at the first named-connection break (concurrent with the label fade-in); from that moment forward, each named-connection break decrements the number by one. Any earlier movement undercuts the reveal — the player would have something to ask about ("what's that counting?") before the design is ready to answer. Mockups, prototypes, and shipped UI must all hold this: any pre-Act-2 surface that displays the number renders it as a literal constant. *(Act boundaries: Act 1 = T1 through T6 Local Group (the peak); Act 2 = T7 Galactic Cluster onward (the decline). A **named-connection break** is an authored narrative event in Act 2 in which a named cosmic structure — like the Eridanus Reach — falls below causal threshold as the universe expands, severing the player from a region they will never reach again. These are narrative beats authored against the cosmological decline, not upgrade losses. The Eridanus Reach pivot at T7 is the first such break, so label fade-in, first decrement, and the Reach falling below causal threshold all fire concurrently at the T7 entry beat.)*
- **Never explain.** The game does not say "you have caused this." The player figures it out from the numbers and the names. The conclusion is unforgettable because the player draws it.
- **Upgrade cards are prose-first.** Every upgrade card shows exactly four content fields: name (real-cosmology term), one-line description (clinical register per the Two-voice UI rule), cost of the next purchase, and current level. No other content. Forbidden, exhaustively: efficiency badges, MPS/MPC/AC-per-second summaries, sort-by-efficiency dropdowns, max-buy or multi-purchase buttons, recommended-next tags, glow-on-the-cheapest, "synergy active" indicators, completion percentage, ROI displays, speed multipliers, queue or auto-buy lists, comparative-best-deal markers, color-coded affordability rankings beyond the single perceptual glow below, long-press tooltips that reveal numerical effects, upgrade-tree visualizers in the shipped game. **Affordability glow carve-out**: a single perceptual "within reach" cue is allowed if and only if (a) subtle warmth, not pulse or blink; (b) identical treatment for every newly-affordable card with no differentiation by cost or "best buy"; (c) not animated past the first-affordable moment. **Order**: author-curated by narrative position, matching gameplay-design.md §3. No player-facing sort controls. **Terminal state**: max-N stackables and purchased one-shots show "Maxed" or "Owned" in the cost slot; cards stay in the sheet until tier transition. **Scope**: binds the upgrade bottom-sheet; does not bind the Settings modal, Inventory artifact, or developer tooling.
- **The narrator confesses, never accuses.** Player guilt comes from being included in the "we", never from direct address.
- **The universe is patient.** Mass accumulates continuously while the player is away. Consolidation does not advance without active purchase decisions — purchase is the only mechanism that converts stored mass into structural progress. The narrator speaks only to a present player; accumulated narrator events held until return (delivered as transient lines on return, not as a readable queue). A player who returns after a long absence is greeted by a quieter line, never the loudest. The narrator does not perform to an empty room.
- **Strategic choice locks at the PEAK tier.** No new strategic mechanics after the peak. No completionist upgrades after the peak. Act 2's content is named-connection breaks, baseline progress, and the CMB through-line. The player witnesses what they built; they do not negotiate with it. The absence of choice in Act 2 IS the emotional content of Act 2. *(Under the 2026-05-13 11-tier renumber, the peak is T6 Local Group; Act 2 is T7-T11. Phrasing is renumber-proof so future ladder reshapes don't require re-vetting.)*
- **Named one-shots are universe events.** Every one-shot upgrade with a real-cosmology name corresponds to a visible structure that appears in the 3D scene on purchase and remains until a named-connection break removes it. Stackables contribute to aggregate scene density (more particles, more glow, more medium) but do not get individual markers. Tiered consolidation upgrades visibly grow per level. The scene is the universe assembling itself in Act 1 and unmaking itself in Act 2. The visual mechanic IS the emotional mechanic.
- **Dark matter is inferred, not rendered.** Dark matter is gravitationally important but not luminous. Any visual representation must work through gravitational signatures (lensing distortion, orbital velocities, gas pooling) rather than direct luminosity. We infer its presence from what flows along it.
- **The cosmological event horizon is the player's true horizon.** "Causal Horizon" (the final-tier name) and "causal threshold" (Eridanus Reach pivot) refer specifically to the cosmological event horizon — what we can ever reach — not the particle horizon (what we've seen). In a Λ-dominated universe these diverge; the design specifically commits to event horizon as the player's true horizon.
- **The CMB cools monotonically.** The CMB through-line registers a monotonically decreasing temperature across the arc. CMB temperature only goes down, never bounces. The hum cools; the light fades; the photon energy density falls below detectability by the final tier (Causal Horizon).
- **Heat death is asymptotic, not an event.** The narrator's late voice implies approach, not arrival. "Goodnight" works because it is the narrator's act, not the universe's; the universe continues, infinitely slowly, after the screen fades.
- **The inversion is narrative truth, not mechanical accident.** The mechanical fact that Completion finishes faster than Threshold by the INVERSION tier is the design's central thesis made measurable. Sim-tuner calibrates tightly at the PEAK tier (~parity), the INVERSION tier (Comp materially faster), and the final tier; the inversion is not a tuning artifact to be smoothed out. **The pre-inversion curve need NOT be calibrated to a tight Comp-vs-Threshold ratio** — the per-tier gap in Act 1 is a *felt opportunity cost* on a single playthrough (the player feels completionist purchases as a real save-time decision without being told it matters), not a *measured asymmetry* anyone observes. The thesis *"the better you play, the faster the universe ends"* lands once, on one playthrough, at the INVERSION tier — not across two compared playthroughs. What's load-bearing is the inversion landing at the INVERSION tier, not the whole pre-inversion curve. *(CD-2 / NEW-1 reframe locked 2026-05-13 — see state-of-play bullet for full rationale.)*
- **Long-burn pacing is the canonical experience.** The canonical experience is weeks of calendar time, not hours of continuous play. Calibration targets are **~5-6 weeks engaged play / ~7-8 weeks casual play / 1-2 months committed casual** for a full Completion path (Threshold runs ~7-8 weeks engaged / ~9-10 weeks casual). Calendar time depends on engagement profile (see state-of-play long-burn lock bullet). Speedrun, idle-skip, and accelerated-time modes are not first-class. Any player-facing "faster path" undermines the thesis — the universe's death takes time. Development tooling is exempt under the existing dev-tooling carve-out — tier-skip, fast-forward, debug speed multipliers, parameter overrides, and other dev/test affordances remain in scope; the rule constrains player-facing speed paths only.

When a proposal touches or would benefit from revising any of these, route it through the `rules-guardian` agent.

## Voice rules (compact)

Full reference: [voice-samples.md](Design%20Documents/voice-samples.md).

- Present tense, even for deep time. The narrator is here, now, watching.
- Specific names whenever possible: real galaxies, voids, structures. Andromeda, Eridanus, NGC 1300, M87, Boötes.
- Numbers are weapons in Act 2, decorations in Act 1. *"Twelve solar masses"* is a flourish; *"6.5 billion solar masses, drifting"* is an obituary.
- Sentences shorten as the game descends. The narrator runs out of breath.
- Two anchor lines to calibrate everything else against:
  - **Tier 5 peak transition** — the most polished line in the game.
  - **The Eridanus Reach pivot** — the second most polished line in the game.

## Working conventions

### Path terminology

A player on the **Threshold path** buys only the upgrades needed to hit the consolidation threshold and advance. A player on the **Completion path** also buys all completionist upgrades (the max-N completionist stackable + completionist one-shot in each tier). "Completionist" remains a property of certain upgrades — the path name is now Completion.

### Folder layout

- `CLAUDE.md` — this file
- `.claude/agents/` — project-scoped agent definitions
- [Design Documents/](Design%20Documents/) — design corpus
- [Simulator/](Simulator/) — `legacy/` (archived spreadsheet artifacts: Python build script, recalc.py, sim_validate.py, .xlsx files) + `playtests/` (active calibration data). Simulator development has moved into [Prototype/](Prototype/) as a JS-in-prototype tool.
- [Prototype/](Prototype/) — T1 HTML + current-state doc + (older) barebones spec; new home for the JS simulator
- `game/` — future home for the React scaffold (not yet created)

### Maintaining this file

When a doc gets a new version, when a workstream's source-of-truth conventions change, or when state-of-play advances, update CLAUDE.md. The `doc-keeper` agent owns this. If you're making changes that would affect any of the above sections, loop `doc-keeper` in.

### Agent roster

Three directors orchestrate by domain; each has access to specialists and project-scoped versions of built-in agents. See `.claude/agents/` for full definitions.

| Director | Domain |
|---|---|
| `creative-director` | Writing, tone, aesthetics, new ideas, the emotional curve |
| `engineering-director` | Simulator, prototype, scaffold, graphics, code |
| `science-director` | Cosmological accuracy, real physics, terminology |

Cross-cutting specialists: `rules-guardian` (load-bearing rules), `writer` (narrator-line generation + audit), `sim-tuner` (simulator workflow + playtest analysis), `doc-keeper` (source-of-truth maintenance), `git-keeper` (session-end commits + push-to-main).

Directors invoke each other directly when crossing domains, and loop the user in for qualitative decisions.

### Git workflow

The project's canonical version-control rule: **at the end of a session, all in-flight work is pushed to `origin/main`.** No long-running feature branches; no stale work-in-flight sitting on a worktree branch indefinitely.

Mechanics:

- Work is often done in a git worktree (e.g., `claude/<adjective-noun-hash>` branches off `main` for isolation). At session end, the worktree branch is merged into `main` (fast-forward preferred), then `main` is pushed.
- The `git-keeper` agent owns this workflow. It stages and commits autonomously, but **always confirms with the user before `git push origin main`** — push is the only irreversible step.
- Trigger is manual: the user signals end of session ("we're done", "wrap up", "ship it", or by invoking `git-keeper` directly). The agent does not auto-fire on session-stop hooks.
- The agent refuses to stage secret-bearing files, refuses force-push to `main`, and surfaces unexpected git state (mid-rebase, mid-merge, conflict markers, detached HEAD) rather than acting.
- Commit messages follow the project's voice rules — concise, no exclamation points, real-cosmology vocabulary when relevant — and include the `Co-Authored-By: Claude` trailer.

See `.claude/agents/git-keeper.md` for the full workflow.
